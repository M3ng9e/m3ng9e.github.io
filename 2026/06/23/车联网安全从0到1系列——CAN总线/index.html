CAN是什么？

趋近于完美的通讯 CAN总线！4分钟看懂！

可以观看上述视频，五分钟讲解了CAN是什么、ECU是什么、CAN的差分信号、CAN帧结构、仲裁优先级。

CAN是什么？CAN总线是什么？

  • CAN的全称是Controller Area Network控制局域网，在CAN这个局域网内，与很多车辆ECU进行通信，且结构为总线型，所以称为CAN总线。

CAN在1986被设计出来的目标是可靠、实时、低成本的通信，但在设计的时候没有考虑安全性，具有无认证，广播，无加密，无源地址的安全缺陷。

CAN的帧结构

学习协议的帧格式，是为了让你理解这个协议在底层是如何运转的，比如如何解决冲突，数据的范围等等。

例如仲裁id为11位，那么范围是000 0000 0000 - 111 1111 1111，也就是0x000-0x7FF的范围。

了解协议的时候，往往需要wireshark抓包再去分析，不然能难去找到对应的数据包，有个小技巧是官网都会有示例的数据包，可以对其进行分析。https://wiki.wireshark.org/CANopen

因为wireshark是软件，所以是硬件处理后的显示。图中可以看到ID段，标注的帧类型，帧长度，数据等等，初学者作为了解即可。

CAN的仲裁机制？

CAN总线是双绞线，具有两条电线，CAN_Low和CAN_High，使用两根线的电压差传递信号。

若有噪音干扰，两条线会同时变化，这就是差分信号，具有抗噪音干扰能力。

仲裁机制是，一个发送显性0，一个发送隐性1时，总线呈为显性。隐性会退出稍后发送。

ID越小，优先级越高。

接触CAN

OBD 口是最直接接触CAN的途径，OBD-II通常位于副驾驶座或驾驶座底下附近。

OBD接口的引脚如图，第6针脚和第14针脚就是CANH和CANL。

通过OBD访问CAN所需的硬件和软件

硬件：你需要一个通过USB连接到OBD-II端口的设备，这样你才能发送和接收CAN数据包。（常见硬件：PCAN-USB Pro FD）

软件：你需要一个既能读写CAN包，也能编码和/或解码CAN包的工具。（常见软件：canutils/zcan pro/canoe）

can-utils

can-utils 是一套 Linux 特有的实用工具，它可以让 Linux 与车辆上的 CAN 网络进行通信，为了发送、接收和分析 CAN 数据包，需要安装 CAN utils。canutils 主要包括 5 个经常使用的工具：

1、cansniffer 用于嗅探数据包

2、cansend 发送一条数据

3、candump 转储所有接收的数据包

4、canplayer 重播 CAN 数据包

5、cangen 随机生成 CAN 数据包

cansend

cansend 是用于将 CAN 帧发送到特定 CAN 接口的工具

cansniffer

cansniffer 是用于嗅探 CAN 数据包的工具。cansniffer 的 -c 参数可以通过颜色高亮突出变化的字节，当需要判断执行某些操作是否会导致 CAN 数据变化时使用。

cansniffer 可以通过仲裁 ID 进行过滤，当需要只显示某一个特定仲裁 ID 的帧时，只需在嗅探的过程中，按减号（-）然后输入 000000，再按 Enter 键清除所有的帧，按加号（+）然后输入仲裁 ID，再按 Enter 键便只显示特定仲裁 ID 的帧。

即使没有钥匙也能启动车辆？

由于CAN是广播，且无认证无加密，所以我们可以有一些针对的攻击方式。
